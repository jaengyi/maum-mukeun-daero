# 05. UI / UX 설계

> Style: Material 3 base, 자체 디자인 토큰 (성장 / 잔디 / 자연 모티브)
> Layout target: 6.0~6.8 인치 안드로이드 폰 (Galaxy 우선)

---

## 1. UX 원칙

1. **하루 30초**: 매일의 기록은 30초 안에 끝나야 한다. 입력 필드 최소화, 디폴트값 적극 활용.
2. **단일 결정**: 홈 진입 시 결정해야 할 것은 단 하나 — "오늘 미션 시작할까?"
3. **즉각 보상**: 행동 후 0.5초 안에 시각·햅틱 보상.
4. **누적의 가시화**: 빈 칸의 부담보다 채워진 칸의 자부심이 보이도록.
5. **막연함 제거**: 항상 "지금 어디쯤", "다음 마일스톤까지 며칠"을 표시한다.
6. **부담 없는 휴식일**: 휴식일도 "오늘은 회복일"이라는 명확한 메시지로 죄책감을 차단.

## 2. 정보 아키텍처 (IA)

```
앱
├── 온보딩 (첫 실행 1회)
│   ├── 환영
│   ├── 신체 정보
│   ├── 능력 측정
│   ├── 운동 가능 요일
│   └── 면책 동의
│
├── 시뮬레이션 결과 (온보딩 직후)
│
└── 메인 (Bottom Nav 3-tab)
    ├── 홈 (오늘의 미션)  ⬅ default
    ├── 성장 (잔디 + 차트)
    └── 계획 (전체 주차 보기)

   ※ 우상단 톱니바퀴: 설정 진입
```

> 3개 탭이 의도된 이유: 사용자의 인지 부담을 최소화. "지금(홈) / 결과(성장) / 미래(계획)"의 시간축으로 자연스럽다.

## 4. 인터랙션 / 모션 가이드

| 동작 | 효과 |
|---|---|
| 세트 완료 탭 | 햅틱 (light), 체크 마크 페이드 인 |
| 미션 완료 | 햅틱 (medium), 잔디 칸 sprout 모션 (0.6s) |
| 마일스톤 달성 | 햅틱 (heavy), 풀스크린 컨페티 + 배지 등장 (1.2s) |
| 화면 전환 | shared element transition (홈 카드 → 운동 화면) |
| 잔디 칸 탭 | scale-press (0.95) + 모달 |

## 5. 잔디 그리드 디자인 디테일

- **셀 크기**: 12dp × 12dp, gap 2dp
- **모서리 둘기**: 2dp
- **색상 단계**: `grass-0` ~ `grass-4` (Theme 토큰)
- **레이아웃**: 가로 12주 (84칸), 세로 7요일 (월~일)
- **현재 주 강조**: 마지막 컬럼에 outline (성장 메인 컬러 1dp)
- **칸 채워짐 기준** (intensity 계산):
  - 0: 운동일이지만 미수행 / 비운동일
  - 1: 운동일 미션의 < 50% 달성
  - 2: 50% ~ 75%
  - 3: 75% ~ 100%
  - 4: 100% 달성 + 평소보다 +20% 초과 수행

## 6. 색상·접근성

- Light 테마 기본, Dark 자동 따름
- 잔디 색상은 GitHub 컨트리뷰션 색상 차용 (눈에 익고 검증됨)
- 색맹 사용자: 잔디 칸 채움 정도가 명도 차로도 구분 가능
- 모든 인터랙티브 영역 최소 48dp × 48dp 터치 영역
- contentDescription 모든 아이콘 / 잔디 칸에 작성

## 7. 카피 가이드 (Voice & Tone)

- **한국어, 친근한 반말 + 격려**
  - "오늘 미션 완료! 잘했어 🌱" (직접 반말 X)
  - "오늘 미션 완료했어요! 🌱" (존댓말 + 가벼운 친근함) ← 권장
- 부정어 회피: "실패", "못 했음" → "다시 해볼게요", "내일 함께 가요"
- 죄책감 유발 X: "왜 안 했어요?" 같은 질책성 카피 절대 금지
- 휴식일 카피: "오늘은 회복일이에요. 근육이 자라는 시간"
