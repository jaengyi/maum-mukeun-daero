# 01. 사업기획서 (Business Plan)

> Project Codename: **MaumMukeunDaero (MMD)**
> Version: 0.1 (MVP 기획)
> Last updated: 2026-05

---

## 1. 비전 (Vision)

> **"누구나 마음먹은 작고 소중한 목표를, 막연함 없이 끝까지 이뤄낼 수 있게 한다."**

사람의 의지는 약한 것이 아니라, **방향성과 피드백이 없을 때** 흔들린다. 마음먹은대로는 사용자에게 두 가지를 제공한다.

1. **명확한 경로** — 현재 수준에서 목표까지 가는 길을 시뮬레이션으로 그려준다
2. **즉각적 피드백** — 매일의 수행이 어떤 의미인지, 다음에 무엇을 해야 하는지 알려준다

## 2. 핵심 가치 명제 (Value Proposition)

| 사용자가 겪는 문제 | 마음먹은대로의 해결 |
|---|---|
| "어떻게 시작해야 할지 모르겠다" | 현재 능력 입력 → 시뮬레이션 기반 개인화 계획 자동 생성 |
| "오늘 뭘 해야 하지?" | 오늘의 단일 Task만 카드 형태로 명확히 제시 |
| "내가 잘하고 있나?" | 잔디 시각화 + 주간 수행률 + 다음 마일스톤까지 거리 |
| "정체된 것 같다" | 정체 감지 시 자동으로 계획 재조정 + 격려 메시지 |
| "그만두고 싶다" | 누적된 잔디(이미 이뤄낸 것)를 보여주어 매몰비용 인식 유도 |

## 3. 핵심 차별화 포인트 (Differentiation)

### 3.1 시중 습관 앱과의 차이

| 항목 | 일반 습관 앱 (Habitica, Streaks 등) | 마음먹은대로 |
|---|---|---|
| 계획 수립 | 사용자가 직접 입력 | **시뮬레이션 자동 생성** |
| 강도 조절 | 고정 | **수행 데이터 기반 동적 조정** |
| 시각화 | 단순 체크 / 캘린더 | **GitHub 잔디 컨셉의 성장 그래프** |
| 동기 부여 | 알림, 포인트, 게임화 | **"내가 자라고 있다"는 시각적 증거** |
| 카테고리 깊이 | 넓고 얕음 | **카테고리별 도메인 지식 내장 (운동 생리학 등)** |

### 3.2 GitHub 잔디 컨셉의 차용

GitHub의 contribution graph가 개발자에게 "꾸준함의 자산"이라는 정체성을 주듯, 마음먹은대로의 **성장 잔디(Growth Garden)**는 사용자에게 동일한 심리 효과를 의도한다. 빈 칸이 보이는 부담보다, 채워진 칸이 만드는 자부심이 크다.

## 4. 타깃 사용자 (Target User)

### 4.1 1차 타깃 (MVP 검증용)

- **연령**: 20~40대
- **상황**: "운동을 시작하려고 하지만, 어디서부터 어떻게 해야 할지 막막한 사람"
- **구체 페르소나**:
  - 김OO, 32세, 직장인 남성, 키 175cm / 80kg
  - 학창 시절 이후 운동을 꾸준히 한 적 없음
  - 헬스장 등록은 부담스럽고, 집/공원 철봉에서 시작하고 싶음
  - 턱걸이 1개도 못 하지만 "10개 정도는 할 수 있는 사람"이 되고 싶음

### 4.2 확장 타깃

- 학습 목표 달성 (자격증, 외국어)
- 식단 / 다이어트
- 독서 / 글쓰기
- 명상 / 자기관리

## 5. 시장 환경

### 5.1 시장 분류

- 1차: **Health & Fitness 앱** (운동 시뮬레이션·코칭)
- 2차: **Productivity / Habit Tracker 앱** (목표·습관 관리)

### 5.2 시장 트렌드 (2024~2026)

- 글로벌 헬스 & 피트니스 앱 시장은 지속 성장 중이며 개인 맞춤 코칭 수요가 큼
- 단순 트래킹 앱은 포화, **개인화된 코칭 + 데이터 기반 피드백**을 결합한 앱이 차세대 트렌드
- AI 기반 운동 코칭 앱 등장(예: Freeletics, Future, Fitbod) — 시뮬레이션·적응형 계획이 차별화 요소
- 한국 시장은 "갓생", "오운완" 등 자기관리 트렌드가 커뮤니티 수준에서 활발

> 시장 규모 등 구체 수치는 사업화 단계에서 별도 리서치로 보강 (Statista, IBISWorld, App Annie 등)

### 5.3 경쟁 앱 비교 (참고)

| 앱 | 강점 | 약점 (우리에게 기회) |
|---|---|---|
| Habitica, Streaks | 일반 습관 트래킹의 표준 | 도메인 깊이 없음, 계획 자동 생성 없음 |
| Freeletics, Fitbod | 운동 계획 자동 생성 | 단일 종목(턱걸이만) 깊은 코칭 부재, 한국어 UX 약함 |
| Strava | 러닝/사이클 커뮤니티 | 일반 목표 관리 불가 |
| 노션, 투두 앱 | 자유도 | 시뮬레이션·생리학 코칭 없음 |

**MMD의 포지션**: "**카테고리별 도메인 지식을 내장한 시뮬레이션 코칭 + 시각적 성장 보상**"의 좁고 깊은 앱.

## 6. 비즈니스 모델 (BM)

### 6.1 단계별 BM 로드맵

| Phase | 기간 | BM | 비고 |
|---|---|---|---|
| **Phase 0: MVP** | ~3개월 | 무료 (본인용 + 베타 사용자) | 검증 단계, 수익 X |
| **Phase 1: Free + Donation** | 3~6개월 | 무료 + 응원 후원 | Google Play 정식 출시 |
| **Phase 2: Freemium** | 6~12개월 | 기본 무료 / 다중 카테고리·고급 분석 유료 | 월 ₩2,900~4,900 |
| **Phase 3: B2B / Coach Marketplace** | 1년+ | 트레이너·코치가 사용자에게 계획 판매하는 마켓플레이스 수수료 | 장기 |

### 6.2 Freemium 기능 분리안 (Phase 2)

**Free**:
- 1개 카테고리 활성 (예: 운동 1개 종목)
- 기본 시뮬레이션 + 기본 잔디
- 30일 데이터 보존

**Premium**:
- 무제한 카테고리 / 동시 다중 목표
- 고급 분석 (정체기 분석, 회복 추천, 월간 리포트)
- 무제한 데이터 보존 + 클라우드 백업
- 테마 / 잔디 색상 커스터마이즈

## 7. 성공 지표 (KPI)

### 7.1 MVP Phase

- [ ] **본인 검증**: 본인이 12주 동안 사용해서 턱걸이 0~3개 → 10개 달성
- [ ] **기능 완결성**: 시뮬레이션 → 일일 트래킹 → 잔디 시각화 → 동적 재조정의 한 사이클 완료
- [ ] **사용성**: 매일 30초 이내 기록 가능

### 7.2 Phase 1 이후

| KPI | 목표 |
|---|---|
| **D7 Retention** | 40% 이상 |
| **D30 Retention** | 20% 이상 |
| **목표 달성률** | 사용 시작 사용자의 30% 이상이 첫 마일스톤 도달 |
| **DAU/MAU** | 30% 이상 (습관 앱은 일일 사용이 핵심) |
| **잔디 채움률** | 활성 사용자의 평균 주간 5일 이상 |

## 8. 위험과 대응 (Risk)

| 위험 | 영향 | 대응 |
|---|---|---|
| 시뮬레이션 알고리즘이 부정확 | 사용자 부상 / 신뢰 하락 | 운동 생리학 자료 기반 보수적 강도 + 면책 고지 |
| 단일 카테고리(턱걸이)로 시장이 좁음 | 성장 한계 | MVP 검증 후 빠른 카테고리 확장 |
| 사용자가 입력을 게을리함 | 데이터 부족 → 피드백 품질 저하 | 30초 안에 끝나는 입력 UX, 푸시 알림 최적화 |
| 안드로이드만 지원 | 시장 절반(iOS) 미커버 | Phase 1 말미부터 KMP(Kotlin Multiplatform)로 확장 검토 |
| 개인 데이터 / 헬스 데이터 보호 | 법적 / 신뢰 이슈 | MVP는 로컬 DB 우선, 클라우드 동기화는 옵션화 |

## 9. 마일스톤

| 시점 | 목표 |

## 10. 한 줄 요약

> **마음먹은대로**는 "막연함 없이, 작심한 목표를 끝까지" 가져갈 수 있도록 **시뮬레이션·피드백·시각화**를 결합한 안드로이드 앱이며, 턱걸이 10개 달성이라는 단일 시나리오로 MVP를 만들어 검증한 후 카테고리를 확장한다.
